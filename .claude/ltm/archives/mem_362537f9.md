# Phase B: Full Entity Type Management

## What was built
- **Backend**: All handler files, DTOs, services, and route wiring already existed. Fixed enum handler test compilation errors (extra args). Added 9 missing RBAC tests (T-C.54-72).
- **UI Router**: Wrapped App in `BrowserRouter`, replaced tab switching with `Routes` for `/entity-types/:id`, `/enums/:id`, and `*` (list pages). Tabs now navigate via `useNavigate()`.
- **Delete Confirmation**: Entity type deletion now shows confirmation modal with entity name.
- **Entity Type Detail Page**: 4-tab layout (Overview, Attributes, Associations, Version History) with full CRUD modals, reorder, version diff comparison.
- **Enum Management**: EnumListPage (create/delete with confirmation) and EnumDetailPage (edit name, add/remove/reorder values, delete with referenced-enum error display).
- **API Client**: Extended with `api.attributes.*`, `api.associations.*`, `api.enums.*`, `api.versions.*` methods.
- **Types**: Added `EnumValue`, `VersionDiffItem`, `VersionDiff`.

## Test coverage
- 137 browser tests, 74 jsdom tests, all Go handler tests
- Browser coverage: 84.64% stmts, 90.96% lines, client.ts at 100%
- Documented exceptions: PatternFly Select dropdown interactions unreliable in Playwright, defensive `!id` guards for impossible states, modal `onClose` X-button handlers (tests use Cancel button instead)

## Key lessons
- `getByText()` in vitest-browser-react uses strict mode — fails when text matches multiple elements (e.g., 'active' matching 'inactive', entity name in heading + description list). Use `getByRole('heading', { name })` or `getByRole('gridcell', { name, exact: true })` instead.
- PatternFly `TextInput type="number"` renders as `spinbutton` role, not `textbox`
- PatternFly `Select` dropdown options render in portals and are hard to interact with in browser tests when populated asynchronously
- `MemoryRouter` with `initialEntries` required for all test renders after adding React Router
