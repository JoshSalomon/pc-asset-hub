## Two API Sets

### Meta API (`/api/meta/v1/...`)
For Admins/Super Admins. Manages entity type definitions, attributes, associations, enums, catalog versions.

Key endpoints:
- `/api/meta/v1/entity-types` — CRUD + copy
- `/api/meta/v1/entity-types/{id}/attributes` — attribute management
- `/api/meta/v1/entity-types/{id}/associations` — association management
- `/api/meta/v1/entity-types/{id}/versions` — version history
- `/api/meta/v1/entity-types/{id}/versions/{v1}/compare/{v2}` — diff
- `/api/meta/v1/enums` — enum CRUD
- `/api/meta/v1/catalog-versions` — catalog version management
- `/api/meta/v1/catalog-versions/{id}/promote` / `demote`

### Operational API (`/api/catalog/{catalog-version}/...`)
For all roles. Scoped to a specific catalog version.

Key patterns:
- `/api/catalog/{cv}/{entity-type}` — instance CRUD
- `/api/catalog/{cv}/{entity-type}/{id}/{contained-type}` — containment traversal
- `/api/catalog/{cv}/{entity-type}/{id}/{contained-type}/{name}` — specific contained entity
- `/api/catalog/{cv}/{entity-type}/{id}/references` — forward references
- `/api/catalog/{cv}/{entity-type}/{id}/references/{ref-type}` — filtered references

Reverse references via query endpoint (higher latency acceptable).
