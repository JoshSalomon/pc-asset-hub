## Roles and Lifecycle Permissions

Access control via OpenShift RBAC (SubjectAccessReview). No custom user database.

### Role Matrix
| Role | Meta API | Operational API | Lifecycle |
|------|----------|----------------|-----------|
| RO | GET only | GET only | None |
| RW | GET only | Full CRUD | Create catalog version (dev), promote dev→test, demote test→dev |
| Admin | Full (non-prod) | Full CRUD | All RW + promote test→production |
| Super Admin | Full (including prod) | Full CRUD | All Admin + demote from production |

### Lifecycle State Machine
```
         RW+              Admin+
Development ──▶ Testing ──▶ Production
     ◀──────      ◀──────────────────
       RW+            Super Admin
```

### Key Points
- Dev mode: configurable mock RBAC for local development outside cluster
- Auth: ServiceAccount token or Bearer token via OpenShift OAuth
- UI hides controls for unauthorized actions (role-based), grays out controls for state-based restrictions with tooltip
