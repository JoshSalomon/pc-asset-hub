## FF-15 Export Plugin Mechanism — Implementation

**Architecture**: Exporter interface + ExporterRegistry + ExportBinding model + PreviewCache.

**Key patterns**:
- `Exporter` interface: Name, Description, ParameterSchema, ValidateSchema, Export. ValidateSchema checks schema at binding creation AND publish preview (catches drift).
- `ExportInput`: Pre-indexed struct with `InstancesByType`, `ChildrenOf`, `LinksByAssoc`. Framework builds this from catalog data — exporters don't walk containment trees.
- `K8sArtifact`: Phase 1 is K8s-specific. YAML string content, not generic []byte.
- `PreviewCache` interface: `InMemoryPreviewCache` for Phase 1. Stores `PreviewCacheEntry` with artifacts + `CatalogUpdatedAt` for optimistic locking.
- Publish two-step: preview (run exports, cache artifacts) → commit (retrieve cache, verify catalog not modified, commit). Without token: fire-and-forget goroutine (30s timeout).
- Download: per-binding, not combined. Different exporters may produce different formats.

**MCP Gateway exporter**: Produces `MCPServerRegistration` + `MCPVirtualServer` CRs (`mcp.kuadrant.io/v1alpha1`). Tool prefix derived from server instance name + "_" (no prefix attribute). Required: `route_name` attribute on servers. Added `virtual_server_type` parameter during implementation.

**Instance names**: Tightened to DNS-1123 (`^[a-z0-9]([a-z0-9-]*[a-z0-9])?$`, max 63). Error message says "Kubernetes resource name" not "DNS label".

**Middleware**: GET /exporters = no auth. Binding CRUD = Admin+. Run = RW+. Download = RW+. Publish preview = Admin+.
