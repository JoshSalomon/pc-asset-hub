## Coverage Discipline (Session 007 Lesson)
- Phase 7 MUST run actual coverage commands and compare against baseline in `docs/coverage-report.md`
- Coverage dropped from 81% to 74.5% because new components were added without proportional tests — only caught because user pushed back
- feat-plan skill updated with HARD-RULE: coverage must not regress below baseline

## Two-Pane Redesign
- Original 3-pane layout (tree + instance list + detail drawer) was confusing — middle pane duplicated the tree's navigation role
- Simplified to 2-pane (tree left + detail right) — tree is sole navigation, detail is inline panel (not Drawer overlay)
- Removed ~300 lines of state/callbacks for the instance list (instanceListType, instances, sorting, pagination, filters, schema loading)
- "Browse Instances" from Overview tab now just switches to tree tab and auto-expands the entity type group

## Flaky T-E.128 Fix
- Root cause was NOT timing — it was mock data returning the same entity type name ("MLModel") for both entity types
- `versions.list` and `versions.snapshot` mocks used static return values instead of `mockImplementation` that distinguishes by entity type ID
- Fix: use `mockImplementation((etId) => ...)` to return different names per entity type
- Lesson: when a test is "flaky", investigate the mock data first — intermittent failures often mask a real bug in test setup

## Makefile for Directory Independence
- All commands should use `make -f /path/to/Makefile <target>` to avoid `cd` issues
- Makefile resolves PROJECT_ROOT via `$(dir $(abspath $(lastword $(MAKEFILE_LIST))))`
- Key targets: deploy, test-backend, test-browser, test-system, test-live, test-all, coverage-backend, coverage-browser
