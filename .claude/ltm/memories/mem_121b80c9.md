## Echo RBAC Pattern: Group-level vs Per-route Middleware

In Echo, there are two approaches to applying middleware:

1. **Group-level** (`g.Use(middleware)`) — applies to ALL routes in the group
2. **Per-route** (`g.POST("/path", handler, middleware)`) — applies to specific routes

For RBAC, the correct pattern is:
- Apply `RBACMiddleware` (which extracts the role from the request) at the **group level** so every request has a role in context
- Apply `RequireRole(minRole)` at the **per-route level** so read endpoints stay open to RO while write endpoints require Admin+

```go
g.Use(middleware.RBACMiddleware(rbacProvider))  // group-level: sets role
requireAdmin := middleware.RequireRole(middleware.RoleAdmin)

g.GET("/entity-types", h.List)                    // all roles
g.POST("/entity-types", h.Create, requireAdmin)   // Admin+ only
```

**Mistake made during Phase A:** Initially only group-level RBAC was applied (setting the role) but `RequireRole` was never used on write routes, allowing RO users to perform write operations. This was caught in the security audit.
