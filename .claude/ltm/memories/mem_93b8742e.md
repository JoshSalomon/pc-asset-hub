## Problem

New UI files (e.g., `LandingPage.tsx`, `useCatalogDiagram.ts`) don't appear in `coverage-final.json` after running `npx vitest run --config vitest.browser.config.ts --coverage`. The `scripts/ui-uncovered-lines.sh` script reports "No files matching ... found in coverage data" and you incorrectly conclude the files can't be tracked.

This happened twice — once with `useCatalogDiagram.ts` and once with `LandingPage.tsx`. Both times it was dismissed as "a V8 limitation with new files" when it was actually a configuration bug.

## Root Cause

The `vitest.browser.config.ts` had `reporter: ['text', 'text-summary', 'json-summary']` — the `json-summary` reporter writes `coverage-summary.json` (aggregate stats only), NOT `coverage-final.json` (per-file line-level data). The `scripts/ui-uncovered-lines.sh` script reads `coverage-final.json`, which was stale from a much earlier run and didn't include the new files.

## Fix

Add `'json'` to the coverage reporters in `vitest.browser.config.ts`:

```typescript
reporter: ['text', 'text-summary', 'json-summary', 'json'],
```

The `json` reporter writes `coverage/coverage-final.json` (note: different directory than `coverage-browser/`). Update scripts to use the correct path.

## How to Verify

After running coverage, check the file is tracked:
```bash
COVERAGE_FILE=ui/coverage/coverage-final.json bash scripts/ui-coverage-check-file.sh <filename>
```

If it shows "NOT FOUND" and lists files that include deleted files (e.g., `OperationalApp.tsx`), the `coverage-final.json` is stale — delete it and re-run coverage.

## Key Lesson

**Never dismiss missing coverage data as "a V8 limitation."** Always verify the coverage file is fresh (check timestamp) and that the correct reporter is configured. New files ARE tracked by V8 browser coverage — the issue is always stale data or wrong reporter config.
