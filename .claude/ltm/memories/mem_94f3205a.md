## CatalogVersion Discovery CRD + Cluster Role Filtering

### Architecture
- **CatalogVersion CRs** are lightweight discovery artifacts created when a catalog version is promoted to testing or production
- **API server** creates/updates/deletes CRs on promote/demote (has DB access)
- **Operator** watches CRs, sets owner references to AssetHub CR, updates status conditions (no DB access)
- **Entity type CRDs** (full schema-as-CRD) are future scope — separate from discovery CRs

### ClusterRole vs Environment
- `environment` (development/openshift) = infrastructure (NodePort vs ClusterIP, RBAC mode, Routes)
- `clusterRole` (development/testing/production) = data visibility (which lifecycle stages API serves)
- These are orthogonal — an OpenShift cluster can be a test deployment

### SanitizeK8sName
- Lowercase, replace non-alphanumeric with hyphens, trim hyphens
- Truncate to 253 chars (K8s DNS subdomain limit)
- Can return empty string → callers must validate
