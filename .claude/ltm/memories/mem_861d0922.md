Type definitions replaced enums as first-class versioned objects (FF-14, branch 011-type-system). 9 base types: string, integer, number, boolean, date, url, enum, list, json. Constraints stored as JSONB on TypeDefinitionVersion. System types (string, integer, number, boolean, date, url) are immutable V1, seeded on startup. Attributes reference type_definition_version_id (replaces type + enum_id). CV BOM pins type definition versions via CatalogVersionTypePin table. API: /api/meta/v1/type-definitions with CRUD + versions. TypeDefinitionResponse includes latest_version_id for UI convenience. AddAttributeModal/EditAttributeModal use td.latest_version_id. AttributeFormFields renders multiline strings as TextArea when constraints.multiline=true. Validation service does NOT yet check type constraints (min/max, max_length, pattern) — see TD-90. Data viewer does not yet render type-aware formatting — see TD-91. UI forms have no inline validation warnings — see TD-92.
