## Problem
Using `vi.stubGlobal('URL', ...)` or `vi.spyOn(document, 'createElement')` in vitest browser tests causes coverage collection to hang indefinitely — both V8 and Istanbul providers. Tests pass but the coverage report never generates.

## Root Cause
Coverage data collection from Chromium back to the vitest process uses `URL.createObjectURL` and DOM APIs internally. Stubbing these globals breaks the coverage pipeline. `vi.restoreAllMocks()` runs at test end, but coverage collection happens *after* the test completes, when the stubs may still be interfering.

## Solution
Don't stub `URL` or `document.createElement` in browser tests that run with coverage. Instead, just verify the API was called without asserting the download mechanism:
```typescript
// BAD — breaks coverage
vi.stubGlobal('URL', { ...URL, createObjectURL: vi.fn() })
vi.spyOn(document, 'createElement').mockImplementation(...)

// GOOD — coverage-safe
renderDetail('Admin')
await page.getByRole('button', { name: 'Export' }).click()
expect(api.catalogs.export).toHaveBeenCalledWith('my-catalog')
```

## Detection
- Tests pass but `coverage-final.json` is never written
- `timeout` command exits with code 124
- V8 log shows tests complete but hangs at report generation
- Works fine with ≤56 tests, hangs with 57+ (the stub test)
- Binary search on test count reveals the exact test
