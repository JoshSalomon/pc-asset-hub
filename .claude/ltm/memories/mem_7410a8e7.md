## Implementation Summary (2026-02-18)

### Features Implemented

**Priority 1 — Core CRUD Gaps:**
1. **EditAttribute** — COW versioning pattern in `attribute_service.go`, uses `attrRepo.Update()` after BulkCopy. Handler: `PUT /entity-types/:entityTypeId/attributes/:name`
2. **RenameEntityType** — Context-sensitive: simple rename when no CVs or only dev CV; deep copy via `CopyEntityType` when testing/production CVs. Two-step API flow with `DeepCopyRequired` error. Handler: `POST /entity-types/:id/rename`
3. **CopyAttributes handler** — Wired existing `CopyAttributesFromType` service to `POST /entity-types/:entityTypeId/attributes/copy`

**Priority 2 — Catalog Version Workflow:**
4. **ListPins** — `GET /catalog-versions/:id/pins` resolves entity type names + versions
5. **ListTransitions** — `GET /catalog-versions/:id/transitions` returns lifecycle history
6. **Stage filter** — `GET /catalog-versions?stage=testing` passes through to existing GORM filter
7. **CV detail page** — New `CatalogVersionDetailPage.tsx` with Overview, BOM, Transitions tabs
8. **Entity selection in CV create** — Enhanced create modal with entity type checkboxes and version display
