## Session 023: Phase 1 Bug Fix & UX Polish Sprint

Branch: `014-bug-fix-ux-polish`, 4 commits, 16 TD items resolved + TD-116 from quality review.

### Key Architectural Decisions

- **TypeDefinitionSelector** extracted as shared component (`ui/src/components/TypeDefinitionSelector.tsx`) — used by both AddAttributeModal and EditAttributeModal. Includes typeahead filter + maxHeight scroll.
- **typeLabel** utility (`ui/src/utils/typeLabel.ts`) — system types show just name, custom types show "name (base_type)".
- **DatePicker timezone fix**: Use `new Date(year, month-1, day)` for dateParse, NOT `new Date(\`${val}T00:00:00\`)` which creates wrong dates in negative UTC offsets.
- **Number float fields**: Use `type="text" inputMode="decimal"` instead of `type="number"` to avoid spinner arrows. Integer fields keep `type="number" step={1}`.
- **List element validation**: Frontend `isValidElement()` mirrors backend `isValidElementType()` for string/number/integer/boolean.
- **Backend integer float literal rejection**: Use `json.NewDecoder` with `UseNumber()` + check for decimal point in `json.Number.String()` to reject `0.0` as integer in lists.
- **Validation sessionStorage**: `useValidation.ts` persists errors in `sessionStorage` keyed by `validation:${catalogName}`. Rehydrates on mount.
- **Regex cache**: Module-level `Map<string, RegExp>` in `validateAttributeValue.ts`. TD-119 tracks unbounded growth risk (low priority).

### Gotchas

- PatternFly `Th` with `sort` prop uses `sortBy.index` matching `columnIndex` — no `aria-sort` attribute until sort is active.
- PF6 Select dropdown renders in a portal — use `document.querySelector` for testing internal structure, not `page.getByRole`.
- `json.Unmarshal` always uses `float64` for JSON numbers. Use `json.Decoder.UseNumber()` to get `json.Number` and inspect the raw string for decimal points.
