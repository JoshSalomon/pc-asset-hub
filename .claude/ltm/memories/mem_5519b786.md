# Key Lessons from Session 001

## Process
- **Never skip approval gates** — present changes and STOP, don't batch phases 1-3
- **TDD for quality review fixes** — even one-line fixes must have a failing test first, user caught me skipping this
- **feat-plan skill is for multi-layer features only** — small UX fixes use TDD directly
- **New endpoints go in architecture.md**, not PRD.md
- **Update docs after implementation refinements** — if implementation evolves beyond plan, update PRD + test plans

## Technical
- **GetContainmentGraph returns duplicate edges** — one per entity type version, deduplicate with map[string]map[string]bool
- **strconv.Atoi("-5") succeeds** — add `|| version <= 0` check for URL path version params
- **Modal error UX** — keep modal open on async load failure, show error inside modal
- **Association roles are perspective-dependent** — outgoing shows target role, incoming shows source role
- **Bidirectional associations are symmetric** — show "references (mutual)" from both sides, never "referenced by"

## Infrastructure
- Kind cluster: name `assethub`, context `kind-assethub`, uses podman (`KIND_EXPERIMENTAL_PROVIDER=podman`)
- After crash: `podman start assethub-control-plane`, wait 10s, may need to delete API pod if CrashLoopBackOff
