## Planning Methodology (`.claude/commands/plan.md`)

Every plan follows mandatory phases with user approval gates:

1. **Phase 1: Architecture Decisions** — research, present decisions. User approval required. Skip only if no arch changes.
2. **Phase 2: High-Level Test Plan** — testing strategy. User approval required (never skippable).
3. **Phase 3: Detailed Test Plan** — specific test cases. User approval required unless Phase 2 approved skipping.
4. **Phase 4: Implementation** — baby steps, bottom-up (DB→Backend→UI), parallel tasks where possible.

### Implementation Rules
- 100% coverage target with documented exceptions per line
- Progress log updated after each step
- Multi-layer features: bottom-up (database → backend → UI)

### Final Steps (in order)
1. Security scan (security-auditor skill/agent), fix vulns, re-run tests, produce report
2. Code cleanup (lint, dedup, docs)
3. Full test run (entire suite + coverage)
4. Documentation updates
5. LTM updates (add new, remove stale)
6. Stage files for commit (document staged and not-staged)
7. Prepare commit message in file
- **NEVER commit or push without explicit user request**
